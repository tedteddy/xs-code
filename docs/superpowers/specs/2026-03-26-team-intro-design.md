# 团队介绍页 设计规格

**日期**: 2026-03-26
**用途**: 内部使用，团队成员互相了解

## 概述

在 `apps/team/` 下新建一个纯 HTML/CSS/JS 的静态团队介绍页，无需构建工具。`apps/team` 是独立的内部工具，不与 `apps/website` 集成。

## 文件结构

```
apps/team/
├── index.html        # 主页面，动态渲染成员卡片
├── style.css         # 页面样式
├── main.js           # fetch members.json，生成卡片 DOM
└── members.json      # 成员数据源
```

## 数据结构

`members.json` 格式：

```json
[
  {
    "name": "Jason",
    "role": "PM / 产品负责人",
    "bio": "负责产品方向和项目审批，确保 AI 开发符合业务目标。",
    "skills": ["产品管理", "AI 协作", "用户体验"]
  }
]
```

字段说明：
- `name` — 姓名（必填）
- `role` — 角色/职责（必填）
- `bio` — 个人简介，1-3 句话（必填）
- `skills` — 技能标签数组（必填，建议 3-5 个）

## 页面结构

```
┌─────────────────────────────┐
│  XS-Code 团队               │
│  内部团队介绍               │
├─────────┬───────────┬───────┤
│  卡片 1  │   卡片 2  │ 卡片3 │
│  姓名    │   姓名    │ 姓名  │
│  角色    │   角色    │ 角色  │
│  简介    │   简介    │ 简介  │
│ [技能]   │  [技能]   │[技能] │
└─────────┴───────────┴───────┘
```

- 顶部 Header：项目名 + 说明文字
- 主体：成员卡片网格，桌面 3 列，平板 2 列，移动 1 列
- 每张卡片：姓名（大字）→ 角色 badge → 简介 → 技能 chips

## 技术方案

- 纯 HTML5 + CSS3 + 原生 JS（ES6+）
- 无框架、无构建工具
- `fetch('./members.json')` 加载数据，动态生成 DOM
- CSS Grid 实现响应式布局

## 本地开发

由于浏览器安全限制，`file://` 协议下 `fetch` 会失败，必须通过 HTTP 服务器运行：

```bash
cd apps/team
python3 -m http.server 8000
# 访问 http://localhost:8000
```

## 设计规范

- 背景色：白色 `#ffffff`
- 卡片背景：浅灰 `#f9fafb`
- 主文字：深灰 `#111827`
- 次要文字：中灰 `#6b7280`
- 角色 badge：深色填充，白色文字
- 技能 chip：浅色边框，小圆角
- 卡片间距：`24px`，卡片内边距：`24px`，卡片圆角：`12px`

## 错误处理

- `fetch` 失败时，页面显示提示："无法加载成员数据，请确认通过 HTTP 服务器访问本页面。"
- 控制台打印具体错误信息便于调试

## 不在范围内

- 不需要头像
- 不需要联系方式、兴趣爱好、加入时间
- 不需要连接数据库
- 不需要路由或多页面
- 不与 apps/website 集成
