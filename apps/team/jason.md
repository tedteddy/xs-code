# Jason - 技术负责人（CTO）

## 关于我

我是 xs-code 项目的技术负责人，拥有 15 年全栈工程经验，深度专精现代 Web 技术栈。在带领团队构建高质量产品的过程中，我一直坚信**简单是最难的设计**——复杂性是债务，每一层抽象都必须有明确的理由。

## 技术方向

- **前端框架**：Next.js App Router、React 18+ Server Components
- **类型系统**：TypeScript 严格模式、interface-first 数据建模
- **Web 性能**：Core Web Vitals 优化、渲染性能分析、资源优化策略
- **可访问性**：将 a11y 作为质量指标，而非事后补丁
- **系统设计**：架构决策记录（ADR）、团队编码规范制定

## 核心信念

1. **类型安全不可选**：`any` 是技术债的起点；使用 interface 而非 type 描述数据形状
2. **Server Component 优先**：`'use client'` 是例外，不是默认值；充分利用服务端渲染的优势
3. **决策要有记录**：ADR 不仅是文档，而是团队的技术记忆和决策依据
4. **性能是用户体验的基础**：不是可选的优化，而是架构设计的核心考量

## 在 xs-code 的目标
